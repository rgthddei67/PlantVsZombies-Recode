# 黑夜随机雨势机制设计

日期：2026-07-20

## 目标

用随机雨势作为黑夜关的专属关卡机制。雨分为小雨、中雨、大雨，雨越大，僵尸获得的动画/轨道位移加速越强，植物获得的正向行动效率加成也越强；大雨额外偶发短促闪电。实现不包含动态光源、阴影、积水、反射或材质 Shader。

## 雨势与倍率

| 雨势 | 僵尸动画倍率 | 植物行动倍率 | 视觉暗幕 | 雨声音量 |
|---|---:|---:|---:|---:|
| 无雨 | 1.00 | 1.00 | 0 | 0 |
| 小雨 | 1.05 | 1.03 | 轻 | 0.18 |
| 中雨 | 1.10 | 1.06 | 中 | 0.28 |
| 大雨 | 1.15 | 1.09 | 重 | 0.45 |

僵尸倍率进入 `Zombie::UpdateAnimSpeed()` 的统一 extra 层：冻结仍优先为 0；否则为 `mExtraSpeed × 减速因子 × 雨势因子`。因此 `PlayTrack` 的 clip 覆盖、不同僵尸的 `mExtraSpeed`、减速和冻结都保持正交。`_ground` 轨道位移读取有效动画速度，故雨势会同步加快步行动画与实际前进，避免滑步。

植物不修改全局 `DeltaTime`，也不整体加速 `Plant::Update()`。在 `Plant` 基类提供雨势行动倍率/行动 delta 辅助方法，仅接入：

- 射手、双发、小喷菇、大喷菇、胆小菇的攻击计时与攻击动画；与生存攻速词条相乘。
- 向日葵、阳光菇的生产计时。
- 阳光菇成长、土豆雷准备、大嘴花消化恢复。

索敌轮询、发光阶段时长、受伤、睡眠、死亡、爆炸清理、贴图检查和卡片冷却不受影响。

## 随机过程

- 仅 `GetBackgroundIsNight()` 为真的背景启用。
- 首场雨在开始战斗后随机等待 20～35 秒。
- 每场雨持续 20～25 秒，雨停后随机等待 35～55 秒。
- 小/中/大雨抽取权重为 50%/35%/15%。
- 暂停时 `DeltaTime` 为 0，天气与粒子共同冻结；生存轮间天气作为场景天气继续自然推进。
- 雨势、阶段剩余时间和大雨闪电倒计时写入关卡存档；旧档缺字段时从无雨和首次等待开始。

## 视觉与光照

- 三档雨丝均复用 `PARTICLE_WHITEPIXEL`，通过 Box 发射器铺满屏幕；`ParticleRotation` 使用先旋转后拉伸的矩阵路径，Position 场令雨丝沿同一风向斜向下穿过战场。
- 小/中/大雨通过生成率、线宽、长度、透明度、速度及大雨双发射器区分；雨丝保持细、低透明、冷蓝并带逐滴亮度差，避免白色竖条。
- `GameScene::DrawWorldOverlay()` 在 GameObjectManager 主体与 UI overlay 之间画蓝灰半透明暗幕，只压暗背景和战场，不压暗按钮与文字。
- 只有大雨维护闪电倒计时，偶发调用低峰值、短持续时间的 `ShowScreenFlash()`；不带真实光源、阴影或反射。

## 声音

- 参考原版 `Challenge.InitLevel()`：暴风雨夜通过 `PlayFoley(FoleyType.Rain)` 启动雨声；`FoleyType.Rain` 绑定 `SOUND_RAIN`，标志为循环播放；`Board.DisposeBoard()` 停止该 Foley。
- 复用原版 `rain.ogg`，按资源键追踪唯一循环声道。每场雨开始时播放，雨停或 Board 销毁时只停止雨声，不影响其他音效。
- 小/中/大雨使用 0.18/0.28/0.45 的请求音量；它仍会乘全局主音量和音效音量。

## AutoTest

新增 `set_weather` 与 `trigger_lightning` 命令，便于固定三档雨势和闪电时机。`dump_state` 增加天气状态、倍率整数投影、剩余时间及僵尸动画 extra 倍率，覆盖：

- 三档视觉差异与暗幕。
- 大雨才允许闪电。
- 僵尸雨势倍率与减速乘算，冻结优先级不回归。
- 攻击植物使用雨势与词条的组合倍率。
- 三档雨声均在播放，切回无雨后循环声道已停止。
