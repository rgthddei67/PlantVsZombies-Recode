# 三线射手设计

## 目标

实现经典三线射手（`PLANT_THREEPEATER`），核心逻辑参考
`D:\PVZ\PlantsVsZombies.NET-master\Lawn_Shared\Lawn\Plant\Plant.cs`，
使用主人确认的真实发射帧，并按主人要求加入非原版边路补偿。

## 行为

- 阳光消耗 325，冷却 7.5 秒，基础攻击间隔 1.5 秒，豌豆伤害沿用普通豌豆的 20。
- 每次攻击检查本行、上一行和下一行；只要任意有效行存在前方可攻击僵尸，就启动整轮攻击。
- 一轮攻击始终发射三颗豌豆：通常向本行、上一行和下一行各发一颗；位于顶行或底行时，越界方向缺少的豌豆改为补发到本行。补偿弹使用 360px/s，本行普通弹保持 290px/s，让同一路的两颗豌豆形成明显间距。
- 棋盘边缘不创建越界行子弹；对应头仍开火，但目标行折回本行形成补偿弹。
- 三个头使用独立子 Animator，并附着在根 Animator 的 `anim_head1/2/3`：
  - `anim_shooting1` 在真实帧 29 发射到下一行；
  - `anim_shooting3` 在真实帧 73 发射到上一行；
  - `anim_shooting2` 在真实帧 111 发射到本行。
- 主人提供的帧号已经是 `AddFrameEvent` 口径，代码直接使用，不再减一。
- 斜向豌豆以目标行为碰撞行，从本株位置发出；初始纵向速度为每原版 10ms
  3 像素，并按原版每 10ms 乘 0.97 衰减，使轨迹自然汇入相邻行。
- 攻击间隔和三个射击动画同时接入现有生存攻速与雨势行动倍率。

## 存档与验证

- 根 Animator 继续由通用存档恢复；三个子 Animator 的完整播放状态和攻击计时器写入植物额外数据。
- 子弹存档增加三线运动标记，读档后继续衰减，而不是退化成恒定斜线。
- `smoke_threepeater.json` 覆盖站位、三头动画、三行出弹、斜向命中，以及顶/底行各自的本行补偿。
- 使用 `clang-playtest` 做日常构建和可见 AutoTest，最后用 `clang-release` 做 LTO 发布验证。
