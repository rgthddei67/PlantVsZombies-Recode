# 植物压扁效果设计

## 目标

- 在 `Plant` 基类提供统一的 `Squish()` 入口，为以后巨人砸击、冰车与投篮车碾压复用。
- 本次不实现三类僵尸及其命中判定，只完成植物侧状态、视觉、存档和 AutoTest 验收。
- 表现参考 C# `Plant.Squish` / `Plant.Draw`：植物留在受击位置并停住当前动画，
  X 方向保持原宽，Y 方向压缩到 0.5。

## 行为

`Squish()` 对预览植物与已经压扁的植物无效。正式植物进入压扁态时：

1. 保存当前视觉位置，停止动画和植物能力，不再承伤或成为僵尸啃食目标。
2. 禁用碰撞、隐藏影子并释放当前格子的植物 ID；压扁残影仍留在原处，
   但同格可立即重新种植，销毁时不会误清掉后来种下的植物。
3. 随机播放原版 `FoleyType.Squish` 使用的 `SOUND_CHOMP` / `SOUND_CHOMP2`。
4. 绘制时以“植物视觉原点下方一个当前地图格高”为底边锚点，将 Y 缩放为 0.5，
   X 保持 1.0。非等比世界缩放由 `Animator` 直接烘入慢路径矩阵和 GPU 实例化记录，
   并递归传给任意层级的附加 Animator；`PauseSubtree()` 同步暂停所有子级播放状态，
   射手头部及未来挂件无需植物子类特判。
5. 主人将原版约 8.33 秒的总时长调短为 5 秒；仍保持末段占总时长 20% 的比例，
   因此最后 1 秒渐隐，计时结束后销毁。

压扁计时只在 `BoardState::GAME` 推进；暂停、生存轮间选卡与胜负结算期间保持画面。

## 存档与验证

`GameInfoSaver` 在植物公共字段中保存压扁标志、剩余时间与冻结的视觉坐标。
读旧档缺少字段时默认未压扁；读压扁档时在派生类额外数据恢复后重新施加暂停、
碰撞禁用、影子隐藏和透明度，避免派生类恢复动画覆盖终态。

新增 AutoTest 命令 `squish_plant`，按 `row/col/index` 选择目标并调用正式入口。
`dump_state` 输出 `squished`、`squishTimeMs`、`animPlaying`、`animFrame`、`alphaPct`，
以及根/射手头部的渲染缩放和头部帧号。`smoke_plant_squish.json` 验证状态、
释放占格、根与子动画停格及 50% 纵向缩放，并在按比例缩短后的 4.5 秒观察点
截图渐隐，最后验证到期销毁。
