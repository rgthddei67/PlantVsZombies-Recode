# 精英舞王僵尸设计 spec

日期：2026-07-21｜状态：已实现并通过可见 AutoTest

## 目标

为黑夜雨天加入稀有精英舞王：只在强台风或超强台风期间由普通舞王概率变异而来，持续向房屋推进、无视植物、每 0.4 秒补召一名普通伴舞并最多维持八名；超强台风下获得额外 70% 动作与移动速度。精英触碰任意小推车时，全场现存小推车直接消失且不触发，精英继续推进。

## 出生与天气语义

- 新类型为 `ZOMBIE_ELITE_DANCER`，独立随机权重为 0，不直接进入持久的出怪池。
- `Board::TrySummonZombie()` 选中 `ZOMBIE_DANCER` 后，在创建前调用唯一的天气变异解析器。
- 仅黑夜背景、当前目标雨势为大雨、台风强度为 `SEVERE` 或 `SUPER`，且本波尚未达到两只精英舞王时有资格判定。
- 每个符合条件的普通舞王出生只 roll 一次，25% 命中后替换为精英舞王；同一波最多两只。扣分仍使用普通舞王的基础成本，保持原波次密度。
- 实际类型一旦生成便永久保持；台风衰减或放晴不还原。当前波成功生成数量进入关卡存档，读档不重 roll。
- `SummonNextWave()` 在推进波次后清零计数；天气切换与台风开始/结束不会重置本波额度。

## 本体行为与数值

- 复用 `DancerZombie::SetupZombie()`，继承 Jackson reanim 已验证的 Die/EatTarget 帧事件、断手、断头、掉头粒子及读档重建，不注册任何新帧事件。
- 本体血量 800。
- 不与植物进入啃食态；仍保留魅惑阵营之间的僵尸互啃规则。
- 不使用普通舞王的响指/定身/四阶段召唤状态机，始终播放月球漫步并沿阵营方向推进。
- 超强台风额外倍率为 1.70，接入 `Zombie::UpdateAnimSpeed()` 的统一 extra 层，与雨势、减速、冻结按既有顺序组合；召唤计时不因该倍率或雨势再次加速。

## 伴舞召唤

- `kMaxActiveBackupDancers = 8`、`kBackupSummonInterval = 0.4f` 均放在 `EliteDancerZombie.cpp` 匿名命名空间，作为主人可直接调整的参数。
- 出生 0.4 秒后首次检查；同阵营存活伴舞不足八名时，每满 0.4 游戏秒补召一名 `ZOMBIE_BACKUP_DANCER`。
- 冻结暂停召唤，减速会按既有僵尸逻辑缩慢计时；暂停时不推进。
- 生成位置循环使用本体周围 3x3 外圈八点：上下同行、前后同排及四个对角。越界行或过于靠近房屋的前方点跳过，继续寻找合法点；顶/底行允许复用其余合法点，仍可维持八名。
- 关联使用八个 EntityManager ID。有效成员必须“仍存在且阵营与精英本体一致”；死亡或被魅惑脱队都释放名额。
- 精英被魅惑时放弃旧伴舞关联；之后补召的伴舞继承魅惑。既有敌对伴舞继续独立行动。
- 召唤计时、八个关联 ID、下一个阵型点及魅惑边沿标志均进入派生类存档。

## 小推车契约

- `Mower` 碰撞回调先询问僵尸是否吞噬全场小推车。
- 普通僵尸保持原逻辑：触发当前小推车并承受 `INT32_MAX` 伤害。
- 精英舞王返回 true：`Board` 遍历全部现存小推车，标记对应行已失车并直接 `Die()`；不调用 `Trigger()`、不播放小推车音效、不伤害精英。
- 清除接口幂等，避免同帧多碰撞重复产生副作用。

## 紫色服装资源

- 新 reanim 为 `Zombie_EliteJackson.reanim`，时间线、剪辑、轨道名与 `Zombie_Jackson.reanim` 完全相同，只把含红色衣物的图像键换成精英资源键。
- `scripts/recolor_elite_dancer.ps1` 用 `System.Drawing` 做逐像素 HSV 转换：只选择红色高饱和像素，将色相映射为深紫，保留 alpha、明暗、布料阴影和抗锯齿；不修改头、头发、皮肤、手套和鞋素材。
- 两个 preset 使用同一脚本生成相同资源；原文件不覆盖。

## AutoTest 与验收

- 新增 `spawn_wave_zombie`，只通过正式天气变异解析器创建测试波次僵尸；`summon_next_wave` 走正式波次推进边沿；`spawn_zombie` 继续保持显式类型直造语义。
- 状态 JSON 暴露：本波精英生成数量、小推车数量、精英当前关联伴舞数、召唤计时、超强台风额外倍率。
- 最小脚本覆盖：普通台风不变异、强/超强台风命中、每波两只上限与新波重置、天气切换不重置、八伴舞封顶、无视植物、超强倍率、冻结/减速组合、魅惑后新伴舞阵营、小推车无触发清空且精英存活、死亡动画正常消失、紫衣截图。
- 完成 `clang-release` 零警告构建；从 `build/clang-release/` 在主人当前桌面可见运行，检查退出码、run.log、状态 JSON、断言和截图。
