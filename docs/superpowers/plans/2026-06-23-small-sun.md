# 小阳光 (SmallSun) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 新增「小阳光」收集物——行为与 `Sun` 完全相同，仅视觉缩放 0.6、价值 15；接到 SunShroom（幼年产小阳光、成熟产普通阳光）；适配存档。

**Architecture:** `SmallSun : public Sun` 复用 Sun 全部逻辑，仅覆盖 scale 与 SunPoint。`Board` 新增 `CreateSmallSun` 系列工厂；`SunShroom::PlantUpdate` 按 `mIsGrown` 分支调用。存档侧给每条 sun 记录加一个 `small` 布尔标志（SmallSun *is-a* Sun，序列化时已被 `dynamic_cast<Sun*>` 捕获，只需补判别标志），读档按标志路由到对应工厂，旧存档默认 `false` 向后兼容。

**Tech Stack:** C++17 / CMake + vcpkg / clang-release 构建 / AutoTest JSON 脚本验证 / nlohmann::json 存档。

**测试约定（本仓库现实）：** 无 xUnit 框架。验证 = `clang-release` 构建（0 warning / 0 error）+ AutoTest 脚本（截图 / dump_state）。存档往返**无法**用 AutoTest 验证（AutoTest 模式短路所有存档读写），故 Task 5 靠构建 + 代码审查验证契约。

- [ ] **Step 3: 写 AutoTest 验证脚本**

新建 `autotest/scripts/smoke_small_sun.json`：

```json
{
  "commands": [
    { "op": "goto_level", "level": 10 },
    { "op": "choose_cards", "cards": ["PLANT_SUNSHROOM"] },
    { "op": "wait_state", "state": "GAME" },
    { "op": "set_sun", "value": 0 },
    { "op": "plant", "type": "PLANT_SUNSHROOM", "row": 2, "col": 1 },
    { "op": "set_timescale", "value": 6.0 },
    { "op": "wait_seconds", "value": 7.0 },
    { "op": "set_timescale", "value": 1.0 },
    { "op": "wait_frames", "value": 2 },
    { "op": "screenshot", "name": "small_sun.png" },
    { "op": "dump_state", "name": "state.json" },
    { "op": "quit" }
  ]
}
```

要点：
- `level: 10` → 夜晚背景（`GameApp.cpp:526` level 10-19 = GROUND_NIGHT），蘑菇不睡（`Shroom::SetupPlant` 仅夜晚不睡），否则白天 SunShroom 睡觉不产。
- `wait_seconds` 是**游戏秒**（scaled），SunShroom 首次产阳光在游戏时间 ~5.55s（mProduceTimer 15→20 共 5s + 0.55s 发光），故 wait 7 游戏秒确保已产出且小阳光仍在（存活 10 游戏秒）。
- `set_timescale 6` 让这 7 游戏秒在 ~1.2s 墙钟内走完，远低于 15s 看门狗。
- `choose_cards` 对未拥有的卡会自动 `AddCard`（`TestDriver.cpp:242`），故 level 10 也能选 SunShroom 起局。

`GameInfoSaver.cpp` 第 470-490 行现为：

```cpp

		float x = s["x"].get<float>();
		float y = s["y"].get<float>();

		Sun* sun = nullptr;

		}
		else {
			sun = board->CreateSun(Vector(x, y), false);
		}

		if (sun) {
			std::string track = s.value("animTrack", "");
			if (!track.empty()) {
				sun->PlayTrack(track);
				sun->SetCurrentFrame(s.value("animFrame", 0.0f));
			}
		}
	}
```

改为：

```cpp
	for (auto& s : j.value("suns", nlohmann::json::array())) {
		float x = s["x"].get<float>();
		float y = s["y"].get<float>();
		int  id = s.value("id", NULL_COIN_ID);
		bool small = s.value("small", false);	// 旧存档无此字段→普通阳光，向后兼容

		Sun* sun = nullptr;
		if (id != NULL_COIN_ID) {
			sun = small ? static_cast<Sun*>(board->CreateSmallSunWithID(Vector(x, y), false, id))
						: board->CreateSunWithID(Vector(x, y), false, id);
		}
		else {
			sun = small ? static_cast<Sun*>(board->CreateSmallSun(Vector(x, y), false))
						: board->CreateSun(Vector(x, y), false);
		}

		if (sun) {
			std::string track = s.value("animTrack", "");
			if (!track.empty()) {
				sun->PlayTrack(track);
				sun->SetCurrentFrame(s.value("animFrame", 0.0f));
			}
		}
	}
```

说明：三目两分支返回类型须一致——`CreateSmallSun*` 返回 `SmallSun*`、`CreateSun*` 返回 `Sun*`，用 `static_cast<Sun*>` 把 SmallSun 上转到公共类型 `Sun*` 后再赋给 `sun`。
