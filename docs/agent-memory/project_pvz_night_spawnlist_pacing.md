# 黑夜第二大关出怪节奏（2026-07-21）

主人试玩发现初版 2-8 同时首次塞入普通橄榄球、舞王和粉色橄榄球，选卡预览与机制学习都过于紧凑；2-9 又机械收录 11 种全部可独立生成僵尸，既挤预览，也因随机池过大稀释重点威胁。当前按绝对关卡 10～18（显示 2-1～2-9）整体重排如下：

| 关卡 | 波数 | 本关定位 | 出怪池 |
|---|---:|---|---|
| 2-1 | 10 | 黑夜基础 | 普通、路障 |
| 2-2 | 15 | 读报教学 | 普通、路障、读报 |
| 2-3 | 15 | 白天旧敌复习 | 普通、路障、撑杆、快速铁桶、读报 |
| 2-4 | 15 | 快速读报教学 | 普通、路障、铁桶、读报、快速读报 |
| 2-5 | 20 | 铁门教学 | 普通、路障、撑杆、铁门 |
| 2-6 | 20 | 普通橄榄球独立教学 | 普通、路障、铁桶、读报、橄榄球 |
| 2-7 | 20 | 舞王独立教学 | 普通、路障、读报、铁门、舞王 |
| 2-8 | 20 | 粉色橄榄球教学/家族对照 | 普通、路障、铁桶、快速铁桶、橄榄球、粉色橄榄球 |
| 2-9 | 30 | 重点机制综合 | 普通、路障、铁桶、快速读报、铁门、橄榄球、舞王、粉色橄榄球 |

关键节奏：2-6 先教普通橄榄球；2-7 完全撤下两种橄榄球，只教舞王/天气变异，且玩家进入 2-7 时尚未取得本关通关奖励毁灭菇；2-8 取得毁灭菇后再教粉色橄榄球，并撤下舞王形成一次无同场复习；2-9 才把三种重点威胁综合。末关保留 `sun: 1500`，但从 11 种降为 8 种，使重点怪更常被正式权重抽中且预览仍可辨认。

两个 preset 的 `spawnlists.json` 规范化 JSON 已统一；顺带把 msvc-debug 里陈旧的 1-7 波数与 1-9 初始阳光同步到 clang-release 权威配置。静态校验覆盖 JSON 可解析、level 唯一、僵尸 ID 无重复/不越界、工厂 gamedata 存在、`weight > 0`、`appearWave <= waves`。

AutoTest 新增 `smoke_night_spawnlists.json`，逐关断言 background、maxWave、完整 spawnList、数量与 2-9 阳光；`dump_state` 新增 `maxWave`/`chooseCardReady` 抓手。clang-release 配置与构建成功；主人当前桌面可见运行观察到窗口标题“植物大战僵尸中文版”，exit 0，35 条 assert 全过，9 份状态 JSON、4 张重点关截图齐全，`run.log` 为 `script finished OK` 且无 FAIL/WATCHDOG。截图确认 2-6 普通橄榄球、2-7 舞王、2-8 普通+粉色橄榄球各自主题清楚，2-9 的 8 种综合阵容仍能排开。

本次方法已固化到 `.agents/skills/adding-zombie/references/adventure-spawnlist-pacing.md`：先画整大关首次登场/复习/综合表；按进入关卡时已有植物评估克制；禁止 `weight: 0` 召唤/变异体入池；双 preset 做规范化内容比较；逐关状态断言并对重点首次关做就绪后截图。
