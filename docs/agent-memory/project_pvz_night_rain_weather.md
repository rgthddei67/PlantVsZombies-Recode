# 黑夜随机雨势天气

## 当前实现（2026-07-21）

黑夜关用间歇式小/中/大雨替代墓碑机制；白天背景不启用。战斗开始后 80～90 秒出现首场雨，新雨首段持续 85～150 秒，切档尾雨持续 45～80 秒，完全放晴后间隔 15～40 秒。初始三档相对权重从前期 45/40/32 平滑变化到后期 34/40/52：普通关在波次进度 40% 后开始以 smoothstep 增强，生存模式再以第 8 轮达到完整后期强度作为轮次下限。初始小雨结束时的中雨/大雨/放晴权重仍为 30/20/45；中雨的“小雨/晴天”由 35/65 变到 15/85，大雨的“中雨/小雨/晴天”由 45/20/35 变到 55/5/40，专门减少后期乏味的小雨尾段。预警也可能得到“天气持续”：晴天续晴权重从前期 18 降到后期 10；一场新雨若首段为中雨/大雨，最多允许同档续一次，权重为 15/12；小雨不续，避免加重后期小雨问题。续期或切档后只允许“大→中/小”“中→小”衰减，尾端小雨必定放晴，故雨链仍然有界。天气强度、阶段剩余时间、大雨闪电计时、小雨增强资格、同档续期资格、过渡来源雨势与过渡余时随关卡存档；瞬态粒子和地面水花不存档，`StartGame()` 按剩余天气重建。

天气预警：每个晴/雨阶段结束前 15 秒预抽取并锁定真实下一天气；公开预报有 75% 概率显示真实结果，另外 25% 只从当前状态机允许、且不同于真实结果的候选中选择。晴天可预报继续晴天或小/中/大雨；中/大雨尚有续期资格时可预报同档持续；无增强资格的尾端小雨下一步必定放晴，因此没有合法错误候选时强制报准。预报与当前天气相同时，牌子明确追加“（持续）”，使“晴天→晴天”成为可验证的真实结果而非一眼可识破的假播报。左上角面板用未缩放时间滑入/滑出，第一行显示当前天气，第二行显示公开预报与揭晓倒计时；天气揭晓后继续显示当前结果 5 秒，随后即使仍在下雨也会收回，直到下一次预警再出现，避免长期遮挡视野。揭晓不一致时在面板下方弹出约 3.2 秒的非模态红色提示，显示“预报 → 实际”，不暂停也不抢输入。公开预报、真实下一天气、ready 标记和当前天气展板的剩余显示秒数均进关卡存档：显示 4 秒后保存，读档只续 1 秒；已经归零并消失的展板不会因仍在下雨而重现。失败提示的剩余时间、错误预报与实际天气也一起保存，读档从原动画进度继续，归零后不重播；只有主面板的滑动进度不存档。

大雨附加台风：每次真正进入新大雨阶段只判定一次，基础概率从前期 25% 随后期因子平滑升到 50%；连续一次大雨未命中后，下次概率额外增加 15 个百分点，最多累计 4 次并封顶 85%，命中后清零。保底次数随关卡存档，读档不重 roll，也不能靠反复读档清掉落空记录。命中后普通/强/超强台风相对权重为 50/35/15。强度只按 `超强台风 → 强台风 → 台风 → 结束` 单向衰减，不在同一场大雨中重新增强；三档衰减前分别维持 18～24、22～30、28～38 游戏秒。风实际吹向每 18～26 游戏秒翻转一次，UI 使用“吹向屋后/吹向前线”与左右箭头实时显示，避免把气象学“来向”误当成游戏位移方向。台风期间天气面板常驻并加宽到 350 逻辑像素，第三行持续显示风向；普通台风显示“持续风（无位移）”，强/超强台风显示阵风倒计时或阶段结束。强度衰减会重新显示 5 秒当前天气通知，普通台风消散后面板恢复原本的预报生命周期。

台风玩法：僵尸自主水平位移先应用大雨 1.40，再乘台风相对倍率；敌对僵尸顺/逆风在普通、强、超强三档依次为 1.08/0.92、1.14/0.86、1.20/0.80，魅惑僵尸因实际移动方向相反而交换顺逆风。台风不改 Animator、攻击或技能计时。普通台风没有阵风位移；强台风等待 12～16 秒、单次 1 格、活动 1.8 秒且每阶段最多 3 次，超强台风等待 14～17 秒、单次 2 格、活动 2.4 秒且每阶段最多 2 次；等待上限均短于对应最短衰减时间，避免自动流程尚未起风就先降档。活动阵风开始时锁定强度与风向，植物结算点一次性预抽在 25%～75% 进度内；衰减与转向计时在这段短阶段暂停，结束后才继续。阵风按风向从前缘逐格结算植物：目标有植物则留在原格，吹入弹坑或越过地图边缘则直接死亡。弹坑本身保留，不能作为挡风墙保护相邻或后方植物。

活动阵风同时对全部存活僵尸施加独立于自主行走的物理漂移：强/超强峰值为 18/32 像素每游戏秒，使用 `4p(1-p)` 包络从 0 平滑升到峰值再回落，单次总漂移约 22/51 像素，明显小于植物 1/2 格。冻结和啃食中的僵尸仍会连同 Transform/碰撞箱一起被吹；魅惑只改变自主行走方向，不改变空气的物理吹向。吹向前线时横坐标钳在画面右缘外 60 像素，低于既有 +65 像素清理线，防止刚出生僵尸被阵风误删；吹向房屋不设额外保护，保留天气威胁。阵风把啃食者吹离目标时由正常碰撞退出链停止啃食。

轻型植物子弹响应实时风向：豌豆、寒冰豌豆以及未来火豌豆按自身 `velocityX` 判断顺逆风；普通/强/超强三档水平速度倍率依次为 1.05/0.95、1.10/0.90、1.15/0.85，命中伤害倍率依次为 1.05/0.95、1.08/0.92、1.12/0.88。孢子、投掷物、重型弹丸、篮球和僵尸豌豆不响应。倍率只在 `Bullet::Update()` 与 `Bullet::BulletHitZombie()` 的公共结算点从当前 Board 台风状态派生，不改写基础速度/伤害，故风向翻转会立即影响在途子弹，放晴自动还原，存档仍只保存基础值。`Bullet.cpp` 的风力档案表必须与 `BulletType` 数量和顺序完全一致；新增或重排枚举而未明确选择风力响应会触发编译失败。

植物换格把逻辑 `row/column`、Cell 占用、Transform 与碰撞箱在随机结算当帧立即切到目标格，再以 0.45 游戏秒纯视觉偏移让本体、阴影和血条平滑追赶。存档只记录已经结算的目标格，不记录视觉偏移；因此滑动中保存/读档会直接落在目标格，不会卡在半格。台风强度、风向、强度衰减余时、转向余时、阵风等待、剩余预算，以及活动阵风的锁定强度/方向、总时长、余时、植物结算余时和已结算标记均入关卡存档；读档继续剩余僵尸漂移，且植物至多结算一次。僵尸实际漂移后的 Transform 本来就在僵尸存档中；风线粒子与植物视觉插值仍属瞬态，读档后按逻辑状态重建或归零。

倍率口径：僵尸为 1.15/1.25/1.40，统一进入 `Zombie::UpdateAnimSpeed()` 的 extra 层，组合式为 `mExtraSpeed × slowFactor × rain`，冻结优先置 0，因 `_ground` 位移读有效动画速度而不会滑步。植物为 1.10/1.15/1.20，只通过 `Plant` 辅助方法加速攻击、生产、成长和恢复；不改全局 delta，也不加速索敌、受伤、死亡、爆炸清理或卡片冷却。攻击速度与生存词条相乘，动画 clip 和计时阈值一起缩放。

视觉：`RainLight/Medium/Heavy.xml` 位于两个 `build/<preset>/resources/particles/config/`，复用 WhitePixel，使用细、低透明、冷蓝、逐滴亮度不同的斜雨丝。小雨为初始 72 粒、每秒 80 粒、alpha 峰值 .25；中雨为 52/60/.25，但通过更粗、更长、更快及更暗的幕保持强度差异，大雨为双层发射器。台风期间大雨改用 `RainHeavyTowardHouse/Front.xml`：吹向屋后以 X=-430/-480、正 28°～37° 形成左斜雨，吹向前线以对称正 X 与负角度形成右斜雨；翻向时 `ParticleSystem::StopEffect` 只停止旧雨继续发射，在途雨丝自然淡出，新方向按剩余雨时立即接管，不能用 `ClearAll()` 误杀爆炸和水花。天气通过正的运行期时长把发射器切成循环模式，`SpawnMaxLaunched` 只作为可复用池容量，不再用累计总配额硬撑整场雨；`Board` 每帧核对当前雨势发射器，若被清理或异常停止会按剩余时间自动补发。地面水花沿用原版 `Rain_splash1～4` 与 `Rain_circle` 贴图，每次在 9×5 草地网格内随机落点播放约 0.3 秒的水花与扩散圆圈；按较大游戏窗口定稿为小/中/大雨平均约每秒 5/10/20+ 次（间隔 0.16～0.24、0.08～0.12、0.02～0.04 秒），落点距网格边缘保留 18px，避免素材跑出屏幕。`ParticleRotation` 显式使用时走“世界中心旋转后再拉伸”的矩阵路径，避免细长贴图角度被压回竖直。暗幕在默认世界粒子之后、UI 之前绘制，统一压暗背景、实体、雨丝、爆炸和命中飞溅，但不压卡片、按钮与文字；当前 alpha 小/中/大为 34/55/110。仅大雨有短促低峰值白闪，无动态灯光/反射。

风线视觉：两个 preset 都有 `WindTowardHouse.xml` 与 `WindTowardFront.xml`，共用原创 `WindStreak.png`，分别从画面右/左边缘外横穿逻辑画面。该 32×256 RGBA 纹理由内置 `imagegen` 在纯品红背景生成单根冷蓝风痕，再按技能流程去色键、验证透明角与半透明边缘后缩图入库；尖端、亮芯和柔和光晕让它在复杂草坪上可见而不再呈硬白直线。每批仍为 10 粒，`ParticleScale [.32 .50]` 与 `ParticleStretch [.75 1.10]` 形成约 61～141px 的随机长度，峰值 alpha .22。三档强度只由 `Board.cpp` 顶部发射间隔 0.30/0.20/0.10 秒控制，因而普通最疏、强台风居中、超强台风最密；风向翻转时立即发射新方向，旧方向粒子在最长 1.15 秒寿命内自然淡出。粒子对象不进存档。

声音：对照 C# 原版 `Challenge.InitLevel()` 的 `PlayFoley(FoleyType.Rain)`、`TodFoley` 中 `SOUND_RAIN` 的 flags=5（Loop + MuteOnPause）以及 `Board.DisposeBoard()` 的 `StopFoley(Rain)`。复用原版 `rain.ogg`，本实现通过 `AudioSystem::PlayLoopingSound/StopLoopingSound` 按资源键唯一追踪循环声道；小/中/大请求音量为 0.20/0.28/0.48，仍乘主音量与音效音量。切档时僵尸倍率、植物倍率、暗幕 alpha 与雨声音量共用 2 游戏秒 smoothstep 过渡，旧雨丝自然收尾并与新发射器短暂重叠；雨转晴时等淡出结束才停止循环声。SDL_mixer 的声道音量会跨播放保留，因此循环状态同时保存 `Mix_Chunk*`：停止、查询和调音前校验声道仍属于该资源，回收后恢复 `MIX_MAX_VOLUME`；普通音效播放也主动复位取得的声道，避免复用低音量雨声声道后随机“吞音效”。
